# WordPress AI Code Checklist

Чек-лист для проверки WordPress-плагинов и PHP-кода, написанного через ChatGPT, Cursor или другие AI-инструменты.

AI может быстро собрать основу проекта, но такой код часто нужно проверять перед запуском на реальном сайте: безопасность, AJAX, SQL, права доступа, структуру файлов и поддержку проекта.

## Что проверять в AI-сгенерированном WordPress-коде

### 1. Безопасность

* Проверяется ли `nonce` в AJAX-запросах
* Проверяются ли права пользователя через `current_user_can()`
* Нет ли прямого доступа к PHP-файлам
* Экранируется ли вывод через `esc_html()`, `esc_attr()`, `esc_url()`
* Очищаются ли входные данные через `sanitize_text_field()`, `sanitize_email()`, `intval()`
* Нет ли возможности выполнить чужой код через форму или параметр URL
* Нет ли открытых AJAX-действий без проверки пользователя

### 2. SQL и база данных

* Используется ли `$wpdb->prepare()` для SQL-запросов
* Нет ли прямой вставки данных пользователя в SQL
* Проверяются ли значения перед записью в базу
* Нет ли дублей записей при повторной отправке формы
* Есть ли индексы в таблицах, если данных много
* Не создаются ли таблицы при каждом открытии страницы

### 3. AJAX и формы

* Правильно ли подключён `admin-ajax.php`
* Есть ли обработчик `wp_ajax_...`
* Нужен ли обработчик `wp_ajax_nopriv_...`
* Возвращается ли ответ через `wp_send_json_success()` или `wp_send_json_error()`
* Обрабатываются ли ошибки на стороне JavaScript
* Не используется ли `$` вместо `jQuery`, если сайт требует строгий режим
* Не ломается ли форма при кешировании страницы

### 4. WordPress-логика

* Используются ли хуки WordPress
* Нет ли кода, который выполняется слишком рано
* Не ломается ли админка WordPress
* Не конфликтует ли код с темой или другими плагинами
* Работает ли код после обновления WordPress
* Не подключаются ли скрипты и стили на всех страницах без необходимости
* Проверяется ли, где именно должен работать код: frontend, admin, AJAX, cron

### 5. WooCommerce

* Не ломается ли корзина
* Не ломается ли checkout
* Проверяются ли статусы заказов
* Работает ли оплата после оформления заказа
* Корректно ли обрабатываются callback/webhook от платёжных систем
* Не конфликтует ли код с доставкой
* Не замедляет ли код оформление заказа

### 6. Производительность

* Нет ли тяжёлых SQL-запросов на каждой загрузке страницы
* Нет ли лишних циклов по всем пользователям, заказам или товарам
* Используется ли кеширование там, где это уместно
* Не создаются ли лишние запросы к базе
* Не подключаются ли внешние скрипты без необходимости
* Не замедляется ли админка WordPress

### 7. Структура проекта

* Понятно ли, где находится основной код
* Есть ли разделение на PHP, JS и CSS
* Нет ли одного огромного файла без структуры
* Понятны ли названия функций
* Нет ли повторов одного и того же кода
* Можно ли безопасно доработать проект через месяц или год
* Можно ли передать проект другому разработчику

### 8. Что часто ломается в AI-коде

* Форма работает в демо, но не работает на хостинге
* AJAX-запрос возвращает ошибку 400, 403 или 500
* Нет проверки прав пользователя
* SQL-запросы небезопасны
* Код работает только в одной теме
* После обновления плагина сайт ломается
* В WooCommerce перестаёт работать checkout
* Код невозможно нормально поддерживать

## Когда AI-код нужно дорабатывать

AI-код можно дорабатывать, если:

* основная логика понятна
* нет критических проблем безопасности
* структура проекта ещё не полностью сломана
* ошибки можно исправить точечно
* проект уже частично работает

## Когда лучше переписать код

Код лучше переписать, если:

* непонятно, как он работает
* одна правка ломает другую часть проекта
* нет нормальной структуры файлов
* много дублей кода
* небезопасно работают формы, AJAX или SQL
* проект нельзя спокойно передать в поддержку
